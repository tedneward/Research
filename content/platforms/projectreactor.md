title=Project Reactor
tags=platform, functional, jvm
summary=A fourth-generation reactive library, based on the Reactive Streams
specification, for building non-blocking applications on the JVM.
~~~~~~

[Website](https://projectreactor.io/) | [Source](https://github.com/reactor)

Several sub-projects:
* [Reactor Core](https://github.com/reactor/reactor-core): A Reactive Streams foundation for Java 8
* [Reactor Test](https://github.com/reactor/reactor-core/tree/master/reactor-test): Test utilities
* [Reactor Extra](https://github.com/reactor/reactor-addons/#reactor-extra): Additional operators for Flux
* [Reactor Netty](https://github.com/reactor/reactor-netty): HTTP, TCP, UDP Clients/Servers using Netty
* [Reactor Adapter](https://github.com/reactor/reactor-addons/#reactor-adapter): Adapt to/from other reactive libraries
* [Reactor Kafka](https://github.com/reactor/reactor-kafka): Reactive bridge to Apache Kafka
* [Reactor Kotlin Extensions](https://github.com/reactor/reactor-kotlin-extensions): Kotlin extensions for all Java artifacts in Dysprosium+
* [Reactor RabbitMQ](https://github.com/reactor/reactor-rabbitmq): Reactive bridge to RabbitMQ
* [Reactor Pool](https://github.com/reactor/reactor-pool): Generic object pool for reactive applications
* [Reactor Core.NET](https://github.com/reactor/reactor-core-dotnet): Incubating Reactive Streams foundation for .NET
* [Reactor CoreJS](https://github.com/reactor/reactor-core-js): Incubating Reactive Streams foundation for JavaScript
